# Catppuccin Porting Patterns Analysis

> Research notes for applying successful theme ecosystem patterns to Violet Void

## Overview

Catppuccin is a highly successful community-driven theme ecosystem with 4 flavors (Latte, Frappé, Macchiato, Mocha) and 26 semantic colors per flavor. This document analyzes their patterns for potential application to Violet Void.

## Key Patterns

### 1. Semantic Color Naming

Catppuccin uses **26 semantic color names** that map consistently across all flavors:

**Accent Colors (14):**
- Rosewater, Flamingo, Pink, Mauve, Red, Maroon, Peach, Yellow
- Green, Teal, Sky, Sapphire, Blue, Lavender

**Base Colors (12):**
- Text, Subtext0, Subtext1
- Overlay0, Overlay1, Overlay2
- Surface0, Surface1, Surface2
- Base, Mantle, Crust

**Application to Violet Void:**
- Consider standardizing semantic names across all ports
- Create a mapping document: Violet Void color → semantic meaning
- Ensures consistency when porting to new applications

### 2. Multi-Flavor Strategy

**Four Flavors:**
- **Latte** - Light theme (high-contrast, daytime use)
- **Frappé** - Dark medium-contrast
- **Macchiato** - Dark medium-high contrast
- **Mocha** - Dark high-contrast (most popular)

**Violet Void Consideration:**
- Currently single dark theme
- Could create variants: violet-void-light, violet-void-soft, violet-void-hard
- Semantic naming would make this easier

### 3. Style Guide Principles

From their style guide:

| Function | Color | Notes |
|----------|-------|-------|
| Background Pane | Base | Primary background |
| Secondary Panes | Crust, Mantle | Sidebar, panels |
| Surface Elements | Surface 0-2 | Cards, inputs |
| Overlays | Overlay 0-2 | Tooltips, dropdowns |
| Body Copy | Text | Primary text |
| Sub-Headlines | Subtext 0-1 | Secondary text |
| Links | Blue | Hyperlinks |
| Success | Green | Confirmations |
| Warnings | Yellow | Alerts |
| Errors | Red | Errors |
| Tags/Pills | Blue | Badges |
| Selection | Overlay 2 @ 20-30% | Text selection |

**Violet Void Mapping:**
```
Violet Void Background → Base
Violet Void Surface → Surface 0
Violet Void Text → Text
Violet Void Accent → Lavender/Mauve
Violet Void Success → Green
Violet Void Warning → Yellow
Violet Void Error → Red
```

### 4. Repository Structure

**Per-Port Repository:**
```
catppuccin-<app>/
├── README.md          # Installation + preview
├── assets/            # Screenshots (.webp preferred)
│   ├── preview.webp   # Generated by catwalk
│   └── ...
├── themes/            # Theme files per flavor
│   ├── latte.theme
│   ├── frappe.theme
│   ├── macchiato.theme
│   └── mocha.theme
└── LICENSE            # MIT (or app-specific)
```

**Naming Convention:**
- Repository: `lower-kebab-case` (e.g., `unreal-engine`)
- Default branch: `main`
- Description format: `<emoji> Soothing pastel theme for <App Name>`

### 5. Automation Tools

**catppuccin/toolbox:**
- **catwalk** - Generates layered screenshots combining all 4 flavors
- **whiskers** - Template-based theme generation
- **palette** - Color palette utilities

**For Violet Void:**
- Create similar tooling for automated port generation
- Template-based approach reduces manual work
- Screenshot automation for READMEs

### 6. Contribution Workflow

```
Idea → Port Request (Discussion) → Draft → Port Review (Issue) → Adoption
```

**Key Points:**
- Template repository for easy port creation
- Community review period before adoption
- Co-maintenance model after transfer
- MIT licensing by default (with exceptions)

### 7. Documentation Structure

**Main Repository (catppuccin/catppuccin):**
- `docs/specs.md` - Palette specifications
- `docs/style-guide.md` - Usage guidelines
- `docs/port-creation.md` - Contribution guide

**Per-Port README:**
- Preview image (catwalk-generated)
- Installation instructions
- Configuration options
- Contributing section

## Recommendations for Violet Void

### Immediate Actions

1. **Create Semantic Color Mapping**
   - Document Violet Void colors with semantic meanings
   - Create `docs/SEMANTIC_COLORS.md`

2. **Standardize Port Structure**
   - Create template repository: `violet-void-template`
   - Define required files per port

3. **Improve Documentation**
   - Add style guide with color usage guidelines
   - Create port creation guide

### Medium-Term

4. **Build Automation Tools**
   - Theme generator from palette definition
   - Screenshot automation for previews
   - Color format conversion (hex, RGB, HSL, etc.)

5. **Consider Flavor Variants**
   - violet-void-soft (lower contrast)
   - violet-void-hard (higher contrast)
   - violet-void-light (daytime variant)

### Long-Term

6. **Community Infrastructure**
   - Contribution templates
   - Review workflow
   - Automated testing/validation
